ETRI Semi Project A팀

## 주의사항

- commit, push 하기 전에 pull 하기
- commit 메세지 규칙
  - 기능 구현 -> `feat:설명`
  - 수정 -> `refactor:설명`
- 브랜치명 규칙
  - 기능 구현 -> `feat/설명`
  - 수정 -> `refactor/설명`
- 작업한 브랜치에서 main 브랜치로 push하고 PR 하기(checkout으로 브랜치 확인 꼭 하기)
- merge 전 다른 팀원 확인 필수
  - 임의로 혼자서 올리면 안됩니다!
